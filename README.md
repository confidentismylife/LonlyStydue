## 一、这个项目要解决什么问题

**一句话**：给定一段行人的历史行走轨迹（过去 3.2 秒），预测他未来会走向哪里（未来 4.8 秒）。

**为什么和火灾有关**：火灾发生时，行人会因为烟气、高温、恐慌而改变正常行走路线。如果能提前预测行人的逃生轨迹，就能动态规划最优疏散路径，减少踩踏和伤亡。

**学术切入点**：现有轨迹预测方法（Social-LSTM、Social-STGCNN 等）只考虑行人之间的社交交互，没有考虑**火灾环境对行人行为的影响**。把烟气扩散数据耦合进行人轨迹预测，就是这个项目的创新点。

## 二、整体设计思路

```
┌─────────────────────────────────────────────────────────┐
│                     系统架构（当前阶段）                    │
├───────────┬───────────┬───────────┬─────────────────────┤
│  数据输入  │  特征提取  │  轨迹预测  │     输出与评估       │
├───────────┼───────────┼───────────┼─────────────────────┤
│           │           │           │                     │
│ ETH/UCY   │  归一化    │ SimpleLSTM│  ADE / FDE 指标     │
│ 行人轨迹  │ ────────→ │ ────────→ │                     │
│ 数据集    │  以最后一  │ 编码器-   │  预测轨迹可视化      │
│           │  帧为原点  │ 解码器    │                     │
│           │           │           │                     │
└───────────┴───────────┴───────────┴─────────────────────┘
```

### 2.1 数据流

一个行人走过摄像头，每 0.4 秒记录一次位置 `(x, y)`。我们把连续的 20 帧切成两段：

```
行人A的完整轨迹: [● ● ● ● ● ● ● ● | ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○]
                   ← 观测 8 帧 →     ← 预测 12 帧 →
                   (模型看到的)      (模型要猜的)
```

- **输入**：8 帧观测轨迹 `(8, 2)` — 即 8 个 (x, y) 坐标
- **输出**：12 帧预测轨迹 `(12, 2)` — 即 12 个未来 (x, y) 坐标

### 2.2 为什么先做归一化

每条轨迹的最后一个观测点作为原点 `(0, 0)`，所有坐标减去这个原点。理由：

- 不同行人的绝对位置差异很大（有人在大门，有人在走廊尽头）
- 模型要学的是**相对位移模式**，不是绝对坐标
- 归一化后所有轨迹从同一个原点出发，模型收敛快得多

### 2.3 评估指标

| 指标 | 含义 | 计算方式 |
|------|------|---------|
| ADE | 平均位移误差 | 预测轨迹和真实轨迹之间，每一帧距离的平均值 |
| FDE | 最终位移误差 | 只看最后一帧（终点）的距离，衡量"目的地"准不准 |

### 3.1 config.py — 全局配置

所有路径和超参数集中管理，其他模块 `from src.config import ...` 引用。好处是调参数时只改一个文件。

```python
OBS_LEN = 8      # 观测 8 帧（3.2 秒）
PRED_LEN = 12    # 预测 12 帧（4.8 秒）
BATCH_SIZE = 32
HIDDEN_DIM = 64
LR = 0.001
EPOCHS = 100
```

### 3.2 data_loader.py — 数据管线

包含三个核心函数，形成完整的数据预处理流水线：

```
原始 txt 文件                    (N, 4) 数组
     │
     ▼ load_eth_ucy_file()
按行人 ID 分组                   {ped_id: [(frame, x, y), ...]}
     │
     ▼ extract_trajectories()
滑动窗口切分                     (N, 8, 2) + (N, 12, 2)
     │
     ▼ normalize_trajectories()
以最后观测点为原点归一化           (N, 8, 2) + (N, 12, 2)
```

**数据格式说明**：ETH/UCY 数据集每个 `.txt` 文件每行为 `frame_id  ped_id  x  y`（tab 分隔），来自 5 个真实场景（ETH 大学门口、Hotel 酒店大堂、Zara1/2/3 商业街、Univ 校园）。

### 3.3 model.py — 模型定义

提供两个模型，复杂度和效果递增：

**SimpleLSTM**（当前使用）：
```
输入 (8, 2)
    │
    ▼ LSTM(hidden=64)
取最后时刻隐藏状态 (64,)
    │
    ▼ Linear(64 → 24)
reshape → (12, 2)
```

- 参数量：约 19,000
- 训练速度：Colab T4 上 100 轮约 5 分钟
- 效果：ADE ≈ 0.38m（比匀速基线好 21%）

**SocialLSTM**（待使用）：
```
输入 (8, 2)
    │
    ▼ Linear(2 → 16)  嵌入层
    │
    ▼ LSTM(hidden=64)
    │
    ▼ Linear(64 → 24) 解码器
reshape → (12, 2)
```

- 多了嵌入层，隐式学到更好的轨迹表示
- 参数量和 SimpleLSTM 差不多，但收敛更快

### 3.4 train.py — 训练流程

```
加载数据 → 归一化 → 划分训练/验证集 → DataLoader
                                            │
                                            ▼
                               ┌─────────────────────┐
                               │  训练循环 100 轮      │
                               │                     │
                               │  每轮做三件事：       │
                               │  1. model.train()   │
                               │     前向 → 算Loss   │
                               │     → 反向传播      │
                               │  2. model.eval()    │
                               │     算 ADE/FDE     │
                               │  3. 保存最佳模型     │
                               └─────────────────────┘
```

关键设计决策：

- **ReduceLROnPlateau**：当验证 ADE 不再下降时自动减半学习率，patience=10 意味着连续 10 轮不进步才调整
- **保存最佳模型而非最后一个**：用 `val_ade < best_ade` 判断，防止过拟合
- **训练/验证 8:2 随机划分**：简单有效，比留一法快

### 3.5 quick_demo.py — 匀速基线

不训练任何模型，直接用物理公式预测：

```
速度 = 最后两帧的位移
预测 = 当前点 + 速度 × 时间步
```

结果是 ADE=0.48m，这是你的**最低及格线**——任何深度学习模型必须比这个好，否则就是白训练了。

## 六、关键设计决策与思考过程

### 为什么先从行人轨迹预测开始，而不是直接做火灾仿真

1. **数据先行**：轨迹预测有公开基准数据集（ETH/UCY），可以立刻跑出结果、写出论文实验章节
2. **分步验证**：先把"纯行人"场景做通，再加火灾特征，能明确归因创新点的效果
3. **风险控制**：FDS 火灾仿真软件的学习曲线高，如果一开始就同时做两个方向，三个月出不了结果

### 为什么不用 Social-STGCNN 的图卷积，而是用 LSTM

1. **代码复杂度**：图卷积需要理解邻接矩阵、拉普拉斯归一化，出 Bug 不好调试
2. **训练成本**：图卷积每个 batch 要构建图结构，显存占用大
3. **渐进式**：LSTM 跑通后，换 Social-STGCNN 只需要改 `model.py` 一个文件，其余管线复用

### 为什么归一化用"最后观测点为原点"而不是 Min-Max

Min-Max 归一化把所有轨迹压缩到 [0,1]，破坏了空间相对关系。以最后观测点为原点保留了轨迹的几何结构，是 Social-LSTM 论文的标准做法。

### 为什么选择 ETH/UCY 数据集

1. **行业标准**：轨迹预测论文 90% 用这个数据集，你的结果可以直接和 SOTA 对比
2. **规模适中**：约 2500 条轨迹，小模型几轮就能看到效果，不会训练一小时没反馈
3. **场景多样性**：5 个场景（大学、酒店、商业街），证明模型的泛化能力
